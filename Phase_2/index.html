Empathize

Memahami Realitas & Psikologi Pasar UMKM

Pada tahap ini, saya melihat dari segi teknis ketakutan dan keterbatasan nyata pemilik UMKM di Indonesia dengan memahami kebutuhan, perasaan, dan motivasi pengguna akhir secara langsung.

Metode yang Digunakan

👁️ Observasi
🎤 Wawancara
🗺️ Empathy Map
📋 Survei

Temuan & Insight

📂 Fragmentasi Data

Data pelanggan UMKM tersebar di WhatsApp, Excel, dan database mentah. Secara bisnis, ini aset berantakan; secara engineering, ini lubang keamanan masif.

💰 Persepsi Biaya Konsultan

UMKM menganggap jasa Data Protection Officer (DPO) manusia adalah kemewahan korporat. Peluang bisnis untuk mendisrupsi harga ini dengan teknologi.

⚠️ Ketakutan Sanksi Administrasi

Kecemasan pemilik bisnis terhadap denda 2% dari pendapatan tahunan menurut UU PDP. Ketakutan ini adalah pendorong utama (buying trigger).

🛡️ Prioritas Kelangsungan Bisnis

Pemilik UMKM lebih takut rekening bank dibekukan karena penyelidikan kebocoran data. Produk diposisikan sebagai "penjaga pintu" operasional.

📉 Keterbatasan Literasi Digital

Kebanyakan pengguna tidak tahu apa itu enkripsi. Saya menyadari untuk tidak menjual "fitur teknis", melainkan menjual "status kepatuhan".

    Define

    Merumuskan Masalah & Peluang RegTech

    Menyaring data empati menjadi strategi produk yang tajam dengan mendefinisikan masalah inti: jurang pemisah antara kewajiban hukum yang ketat dan kemampuan teknis UMKM yang minim.

    Metode yang Digunakan

    📌 Point of View
    🙋 How Might We
    🔗 Affinity Diagram
    🧩 User Persona

    Definisi Masalah

    🌉 Compliance Gap

    Jurang pemisah antara kewajiban hukum pemerintah yang ketat dan kemampuan teknis UMKM yang minim.

    🎯 Penentuan Ruang Lingkup Data

    Fokus utama bot didefinisikan untuk melindungi NIK, nomor telepon, dan data transaksi—tiga elemen yang paling sering disalahgunakan.

    💬 Rumusan Masalah Utama (HMW)

    "Bagaimana cara mengotomatisasi pengawasan data 24/7 tanpa menambah beban gaji karyawan atau biaya server yang mahal?"

    💎 Penetapan Value Proposition

    LexGuard AI sebagai "DPO Digital Mandiri". Bisnisnya adalah kepatuhan hukum, teknologinya adalah otomasi siber.

    ⚖️ Analisis Risiko Kegagalan

    Risiko terbesar jika bot dianggap sebagai beban tambahan bagi server. Definisinya harus "ringan dan non-intrusif".

      Arsitektur Solusi & Skalabilitas Bisnis

      Menggunakan kreativitas engineering untuk membangun model bisnis menguntungkan (skalabel) dan arsitektur solusi yang teknis aman.

      Metode yang Digunakan

      🌩️ Brainstorming

      Ide Utama

      🦙 Arsitektur Local-First AI (Ollama)

      DeepSeek via Ollama. Menjamin kedaulatan data tidak keluar server klien, dan menghilangkan ketergantungan API pihak ketiga.

      ⚙️ Orchestration Alur Kerja (n8n)

      n8n sebagai "mesin audit otomatis" untuk validasi enkripsi dan laporan tanpa campur tangan manusia.

      📄 Sistem Pelaporan ROPA Otomatis

      Sistem menghasilkan dokumen Record of Processing Activities otomatis dalam format PDF sebagai fitur "pembunuh" saat audit.

      👁️ Deteksi Anomali Berbasis Perilaku

      AI mendeteksi ekspor data massal yang mencurigakan sebagai perlindungan dari pencurian data internal.

      💳 Model Monetisasi SaaS Tiered

      Skema paket gratis untuk pemindaian dasar, dan paket pro untuk audit mendalam serta notifikasi pelanggaran 72 jam.

        Prototype

        Membangun MVP yang Tangguh

        Tahap pembuktian desain MVP secara nyata menggunakan modern stacks yang efisien agar siap diuji di dunia nyata.

        Teknologi Stack

        ▲ Vercel
        🗄️ Turso
        ⚙️ n8n
        🦙 Ollama

        Prototipe Dashboard

        ▲ Dashboard Minimalis di Vercel

        Indikator visual: Hijau berarti aman/patuh, Merah berarti ada celah risiko.

        🗄️ Infrastruktur Turso (Always-On)

        Permanent uptime, sehingga log audit kepatuhan bisa diakses kapan saja oleh regulator tanpa kendala cold start.

        🔗 Node Integrasi n8n Adaptif

        Konektor yang bisa dihubungkan ke berbagai database umum (Postgres, MySQL, Sheets) dalam hitungan menit.

        🧠 Integrasi Model LLM Lokal

        Ollama memproses data sensitif dan memberikan rekomendasi perbaikan instan di dashboard.

        ⚠️ Simulasi Notifikasi Pelanggaran

        Sistem peringatan dini melalui WhatsApp/Email jika ditemukan data pribadi yang "telanjang" (tidak terenkripsi).

          Hasil Akhir

          🔓 Uji Penetrasi Sederhana

          Memasukkan data NIK dummy ke database untuk memvalidasi n8n dan AI berhasil menangkap dan melaporkannya.

          📊 Survei Willingness-to-Pay

          Mempresentasikan dashboard kepada startup founders: "Apakah laporan ini berharga sehingga Anda bersedia langganan?".

          👁️ Evaluasi UI/UX bagi Non-Teknis

          Mengamati orang awam mengoperasikan dashboard, menyederhanakan logika interface jika mereka bingung.

          📉 Analisis Efisiensi Biaya Operasional

          Menghitung total biaya Vercel + Turso + n8n untuk memastikan biaya di bawah harga jual.

          ⚖️ Iterasi Berbasis Regulasi

          Menguji laporan agar sesuai dengan format panduan dari Kementerian Kominfo/Badan Pengawas PDP.
